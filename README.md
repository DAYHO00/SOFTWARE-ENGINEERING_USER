## 🔹 과제 0. 개인 과제 (도플 보고서)

### 개요
개인 단위로 수행한 과제로, 소프트웨어공학 관련 개념을 분석하고 정리한 **개인 보고서**입니다.

### 주요 내용
- 소프트웨어공학 개념 정리
- 문제 정의 및 분석
- 개인 관점의 해석 및 정리


---

## 🔹 과제 1. 팀 프로젝트 – 도구 분석

### 개요
팀 프로젝트로 진행된 과제로, **소프트웨어 개발 및 협업에 사용되는 도구들을 선정하고 분석**하는 과제입니다.

### 주요 내용
- 협업 도구 분석  
  - Microsoft Teams  
  - Slack
- 정적 코드 분석 도구 분석  
  - SonarQube  
  - AI 기반 코드 분석 도구
- 도구별 특징, 장단점, 활용 시나리오 비교


---

## 🔹 과제 2. 팀 프로젝트 – 프로젝트 기획 및 분석

### 개요
실제 서비스(OTT 스트리밍 서비스)를 대상으로 **문제 정의(Pain Point)를 도출하고 개선 방향을 기획**한 과제입니다.

### 주요 내용
- 사용자 관점 문제 정의 (Pain Point 분석)
- 기능적 / 비기능적 요구사항 도출
- 목표 시스템 정의
- 핵심 기능 설계

---

## 🔹 과제 3. 팀 프로젝트 – WBS 및 일정 관리

### 개요
프로젝트를 체계적으로 수행하기 위해 **WBS(Work Breakdown Structure)와 프로젝트 일정 관리 체계**를 설계한 과제입니다.

### 주요 내용
- 프로젝트 단계별 작업 분해
- WBS 기반 일정 수립
- 마일스톤 정의
- 프로젝트 라이프사이클 설계

---

## 🔹 과제 4. 팀 프로젝트 – 최종 프로젝트

### 개요
앞선 과제들을 종합하여 수행한 **최종 팀 프로젝트**입니다. 기획, 설계, 일정 관리, 품질 관리까지 소프트웨어공학 전반을 반영하였습니다.

### 주요 내용
- 요구사항 정의 및 설계
- UI/UX 개선안 제시
- 품질 관리 전략
  - 동료 검토
  - 제품 메트릭 수집
  - 정적 코드 분석
- 변경 관리 및 운영 전략
- 프로젝트 전체 흐름 정리
